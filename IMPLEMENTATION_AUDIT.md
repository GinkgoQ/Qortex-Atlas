# Qortex 1-4 implementation audit

Status is based on current executable code, not documentation or the proposal images.

- **Implemented**: production path exists in Qortex and is connected to Atlas.
- **Partial**: useful real implementation exists, but one or more required computations, contracts, or UI workflows are absent.
- **Core only**: Qortex implements the reusable capability, but Atlas has no connected product workflow.
- **Missing**: no adequate production path was found.

## Cross-cutting defects found during audit

1. `/models` used a three-entry Atlas-only compatibility list rather than Qortex's public model zoo. One entry was an unpublished custom architecture. The endpoint now exposes the registered public zoo, contracts, sources, licenses, runtime state, and persisted cache evidence; compatibility remains explicitly separate.
2. Backend availability crashed when an optional runtime was installed but broken for reasons other than `ImportError`. Runtime probing now reports the backend unavailable with the exact diagnostic.
3. Atlas had no model-zoo workspace. It now has a filterable, bounded registry view, live backend probes, exact cache-manifest state, source links, license evidence, runtime state, and offline availability.
4. The proposal image's `ds000001` participant counts are illustrative, not current dataset evidence. Live verification returned 16 rows, 15 valid sex values, and one rejected `M,` value at row 5; Atlas now renders those actual values and never copies the image's 17-participant figure.
5. The existing `scripts/run_real_zoo_inference_demo.py` labeled randomly initialized Braindecode architecture outputs as real predictions. It now runs a pinned pretrained MONAI checkpoint against pinned public BraTS 2023 MRI volumes and records the exact input/model hashes, revisions, licenses, timings, metrics, and NIfTI artifacts.
6. The MONAI adapter crashed on current string-valued bundle task metadata, missed nested `Activationsd(sigmoid=True)`, and incorrectly applied argmax to overlapping BraTS output regions. It now accepts current metadata and preserves independent thresholded TC/WT/ET channels.
7. `msd_brain` imported `DecathlonDataset` from the wrong namespace and treated MONAI-loaded tensors as paths. The loader now uses `monai.apps`, preserves validated paths, validates split/bounds, and correctly exposes stacked 4-D modality data as channel-first arrays.
8. Repeated random access into a compressed `.nii.gz` made whole-run DVARS effectively quadratic because each frame seek restarted gzip decompression. The QC report now performs one memory-bounded sequential block read; the real 208-volume ds000117 run fell from more than 90 seconds without completion to 2.518 seconds.
9. Pearson connectivity hardcoded every input as EEG and graph serialization omitted degree/strength arrays. Signal type and node semantics are now explicit parameters, and the complete per-node graph evidence is serialized.
10. Nine downloaded ds000117 FIF paths existed; direct MNE inspection found one intact recording and eight incomplete files. The real analysis endpoint attempts parse validation, records failures, and selected the intact public recording after rejecting four earlier candidates in manifest order.
11. The proposal treats NIfTI and NumPy as conversion outputs, but Qortex only had registered production writers for Parquet, Zarr, HDF5, WebDataset, Hugging Face, and TFRecord. Atlas now exposes only installed registered writers and states why NIfTI/NumPy are not available. A real ds000117 events conversion produced 95 samples and three Parquet shards through the HTTP/UI workflow; a truncated public FIF verified strict failure cleanup without publishing a partial artifact.
12. Atlas's job registry was process-local and therefore could not serve as scientific experiment history. Persistent Runs now reconstructs only readable artifact-backed runs across restarts. The newest public BraTS run records synchronized CUDA inference at 0.501 seconds, 3,107,262,976 peak allocated bytes, the active RTX 4090 Laptop GPU/software environment, and SHA-256 hashes for every binary output; older runs are retained but visibly marked when environment or artifact-hash evidence was not captured.
13. The validator normalizer preferred an absolute `file.path` over the validator's dataset-relative path and leaked host paths into issue rows. It now prefers `relativePath` and bounds absolute paths against the validated root. A real local ds000001 run with bids-validator 1.15.0 completed in 0.393 seconds, returned one warning at `participants.tsv`, and proved only 5 of 136 immutable-manifest files were present; Atlas therefore labels the verdict “valid local content,” not “valid snapshot.”
14. The range-byte cache alone could not explain compressed NIfTI warm performance because the second request bypasses byte lookup through the decoded-volume LRU. The two cache layers now have separate counters. Repeating the real ds000001 T1w slice measured a 3.892-second cold decode with 5,663,237 inserted bytes, followed by a 0.010-second decoded-volume hit with 11,796,480 resident decoded bytes; the report records one miss and one hit rather than attributing the speedup to a fabricated range-cache hit.
15. The Torchvision adapter resolved constructors only from the top-level `torchvision.models` namespace, so every registered detection architecture under `torchvision.models.detection` failed before inference. It now resolves through Torchvision's registered model-builder API, preserves the selected weights enum, uses its preprocessing and COCO categories, and returns semantic class names. A pinned COCO val2017 run with Faster R-CNN ResNet-50 FPN v2 produced 23 thresholded predictions against 19 non-crowd targets: 8 TP, 15 FP, 11 FN, precision 0.3478, recall 0.4211, and mean matched IoU 0.8284. The report states that these are single-image verification metrics, not dataset mAP.
16. The model cache's `remove()` operation only dropped manifest provenance and had no safe artifact lifecycle; exposing it directly would either leak disk usage or invite arbitrary recursive deletion. The public control surface now classifies storage ownership structurally, rejects unknown roots/symlinks/shared targets, verifies the recorded SHA-256 against the file or bundle before mutation, moves the exact target to Qortex trash, persists the complete original cache entry in a recovery receipt, and rolls back the move if receipt/manifest persistence fails. The real 18.0 MB MONAI bundle and 167 MB Torchvision checkpoint were inspected as removable but were not deleted during verification.
17. Atlas's shared `tinyTable()` renderer indexed rows only by column-header strings even though several call sites supplied ordered arrays, so demographic and newly added comparison/source tables rendered blank cells. It now supports both keyed objects and ordered rows. Live `ds000001@1.0.0` versus `ds000117@1.1.0` comparison rendered all rows: age mean difference -2.8125 years, 95% CI [-4.9040, -0.7210], Hedges g -0.9465, raw p 0.01009, BH p 0.02018; sex Fisher p 0.2852 and Cramer's V 0.2301 after excluding the exact dirty `M,` value at ds000001 row 5. These statistics describe dataset membership and do not claim a causal or clinical group effect.
18. The Quality route passed a nullable tab argument into `tabQuality`, whose default parameter only covered `undefined`; opening a dataset's Quality tab therefore failed while reading the readiness modality. The route now normalizes a null query to an empty object. A live Chrome run loaded `ds000117@1.1.0`, set DVARS to 25.0, and persisted all 208 volumes: 36 flagged, 172 retained, 2.646 seconds. The downloaded mean volume is finite float32 NIfTI with shape 64 x 64 x 33 and SHA-256 `d7333b5439ef4d7908db7cae9fb6478acd9d44008541eb9865f8bd23b054eaa7`; Atlas exposed mean-volume, framewise-table, scrub-plan, and provenance links without browser errors.
19. The readiness endpoint accepted a modality but computed `n_recordings`, `n_label_ready`, and subject evidence over the unfiltered manifest, never passed the existing `LabelPolicy`, never inspected locally downloaded event tables, and only returned a hardcoded split suggestion. Readiness and `can_train` now share modality-scoped recordings, execute an explicit local events-column/missing-value policy, and derive split-group feasibility from recording entities. Live `ds000117@1.1.0` with modality `meg`, label column `stim_type`, missing policy `error`, and subject split returned 794 scoped recordings, 288 label-ready recordings, 17 independent subject groups, and `possible`; Atlas rendered the same decision in Chrome.
20. Applying a standard-space atlas directly to the locally available raw ds000117 BOLD would have produced plausible but scientifically invalid ROI values. The ROI workflow instead requires the public development-fMRI image whose filename declares `space-MNI152NLin2009cAsym`, hashes that BOLD image and its reduced-confounds table, resamples the Schaefer-100 atlas with nearest-neighbor interpolation, and rejects unexpected label coverage. The live full 168-frame run retained 167 frames after the real 0.5 mm FD rule, produced 2,231 edges at absolute r >= 0.3, density 0.4507, one connected component, and six downloadable artifacts. Chrome rendered all 100 ROI rows, the 100 x 100 matrix, a 1,793-pixel montage, and the interactive NIfTI overlay without errors.
21. Coverage subject sorting compared integer natural-sort components from numeric subject IDs with string components from `sub-emptyroom`, crashing ds000117 coverage before rendering. Natural-sort components are now type-tagged. More importantly, the new design contract keeps observed-only absence neutral until exact selectors and expected subjects are submitted. A live ds000117 Chrome declaration for the first recording definition across all 17 visible subjects produced one available and 16 missing cells under contract `8db353752d7f...`; the UI exposed Available, Missing, Not expected, and Unexpected available separately.
